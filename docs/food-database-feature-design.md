# 饮食库功能设计文档

## 功能概述

开发「饮食库」功能，作为 AI 饮食记录解析的辅助数据库，提供精确的营养成分数据支持，并支持用户自定义管理。

## 核心需求

1. **AI 解析辅助**：在用户输入饮食记录时，从饮食库中匹配食物并提供选择
2. **智能数据收集**：从用户输入的详细营养信息中提取并添加到饮食库
3. **数据管理**：提供完整的增删改查界面
4. **数据迁移**：支持批量导入导出功能

## 核心交互规则

### 1. 饮食库匹配规则

- 仅对用「」包裹的食物名称进行饮食库匹配
- 未用「」包裹的文本视为普通描述，不进行任何处理
- 匹配发生在用户输入阶段，而非提交后解析阶段

### 2. 智能添加触发规则

- 检测「食物名称」+（营养成分）的组合模式
- 系统自动识别（）内是否包含数字和营养成分关键词
- 满足条件时显示显式添加提醒

## 详细交互设计

### 场景 1：复制粘贴完整营养信息

**用户输入示例：**

```
「鸡肉肠」25g（每百克542千焦，12.6g蛋白质，2.8g脂肪，13.2g碳水）
```

**系统行为：**

1. 检测到「」+（）模式
2. 分析（）内容，识别出营养成分数据

4. 用户点击后弹出确认对话框，预填充解析的数据
5. 用户确认后添加到个人饮食库

## 数据模型

### FoodItem 数据结构

```typescript
interface FoodItem {
  id: string;
  userId: string;
  name: string; // 从「」中提取的食物名称
  category?: string; // AI生成的分类标签
  nutritionPer: number; // 营养成分对应的数量（如100）
  nutritionUnit: string; // 单位：'g', 'ml', '份', '个'等
  nutrition: {
    calories: number; // 卡路里/千焦
    protein: number; // 蛋白质(g)
    fat: number; // 脂肪(g)
    carbs: number; // 碳水化合物(g)
    // 可扩展其他营养成分
  };
  sourceText: string; // 原始输入文本，用于用户参考
  usageCount: number; // 使用次数统计
  createdAt: Date;
  updatedAt: Date;
}
```

### 解析上下文

```typescript
interface ParseContext {
  hasQuotedFoods: boolean; // 是否包含「」食物
  quotedFoods: string[]; // 提取的「」食物列表
  hasNutritionData: boolean; // 是否包含营养成分数据
  nutritionSegments: NutritionSegment[]; // 营养成分片段
}

interface NutritionSegment {
  foodName: string; // 从「」提取的食物名称
  rawText: string; // （）内的原始文本
  parsedNutrition?: ParsedNutrition; // 解析后的营养数据
  canAddToLibrary: boolean; // 是否可添加到饮食库
}
```

## 功能模块设计

### 1. AI 解析集成

- 在现有饮食记录输入流程中增加饮食库匹配步骤
- 提供候选食物列表供用户选择
- 用户可选择使用库数据或保持原文

### 2. 智能添加功能

- 检测用户输入中的详细营养信息
- 显式提示用户是否添加到饮食库
- 引导用户完善食物信息（名称去重、分类确认）

### 3. 数据管理页面

- 食物列表展示（支持搜索、分类筛选）
- 添加/编辑/删除食物
- 批量操作（删除、导出）

### 4. 导入导出功能

- JSON 格式数据导出
- 支持导入时的数据验证和去重处理

## UI/UX 设计要点

### 1. 输入框增强

- 实时检测「」和（）模式
- 悬浮提示框设计，不干扰正常输入
- 键盘友好的交互流程

### 2. 饮食库匹配界面

- 仅在检测到「食物名称」时显示匹配选项
- 匹配结果以下拉或侧边栏形式展示
- 用户可快速选择或忽略

### 3. 添加确认界面

- 预填充解析的营养数据
- 允许用户修改食物名称和分类
- 显示营养成分的单位和数值
- 提供"下次不再提醒此类食物"选项

## 技术实现要点

### 1. 文本解析引擎

- 正则表达式匹配「」和（）模式
- 营养成分关键词识别（卡路里、千焦、蛋白质、脂肪、碳水等）
- 数值和单位提取算法

### 2. 实时交互处理

- 输入事件监听（keydown, input）
- 悬浮框位置计算和显示逻辑
- 键盘事件处理（Enter 确认、Esc 取消）

### 3. 饮食库匹配算法

- 模糊匹配算法（考虑同义词、简写）
- 匹配结果排序（使用频率、相似度）
- 缓存机制优化查询性能
